## 1. 権限レベルの概要

SportEaseでは、`root`、`admin`、`student` の3つの基本権限を利用します。クラス単位の操作対象は、プロフィールで設定された `class_id` から決定します。

| 権限レベル | 説明 | 対象ユーザー |
|-----------|------|-------------|
| **Root** | システム管理者。全ての機能にアクセス可能 | システム運営者 |
| **Admin** | イベント運営者。Admin機能とStudent機能にアクセス可能 | 教員・スタッフ |
| **Student** | 一般ユーザー。学生向け機能にアクセス可能 | 生徒 |

### 権限の取得方法
- **Root/Admin/Student**: Rootユーザーが「権限管理」画面から `admin` / `root` を付与・剥奪します。`student` は基本ロールとして付与されます。
- **クラス所属**: 初回ログイン時に選択したクラスをプロフィールへ保存します。プロフィール完了後は一般ユーザーによる変更はできません。

## 2. Root専用機能

Root権限を持つユーザーのみがアクセスできる機能です。

### 2.1 大会情報登録・管理
**パス**: `/dashboard/root/event-management`

**機能概要**: スポーツ大会（イベント）の作成・編集を行います。

**使用方法**:
1. サイドバーの「Root」セクションから「大会情報登録・管理」をクリック
2. 「新規大会作成」ボタンから新しい大会を登録
3. 既存の大会をクリックして編集・削除

**設定項目**:
- 大会名
- 開催日時
- 開催場所
- 大会のステータス（準備中/予定/開催中/アーカイブ）
- 2競技への重複登録を許可するクラス人数（初期値：31）

**重複登録のルール**:
- 原則として、選手は同一大会で1競技まで登録できます。
- 設定した人数以下のクラスは、選手1人につき2競技まで登録できます。
- 専攻科・教員チーム（`専教`）は重複登録の上限対象外です。
- この設定は大会ごとに保存されます。クラス人数設定で登録した在籍人数に合わせて設定してください。

**大会ステータスの運用**:

| ステータス | 利用目的 |
| --- | --- |
| 準備中 | 競技登録、チーム編成、トーナメント生成、昼競技設定などを行う状態です。現在操作する大会として選択されます。 |
| 予定 | 開催前の大会情報を保管する状態です。 |
| 開催中 | 大会当日の運用状態です。試合結果と昼競技結果を入力できます。 |
| アーカイブ | 終了した大会の参照用状態です。 |

**パス**: `/dashboard/root/rainy-mode`

**機能概要**: 雨天時の代替競技スケジュールへの切り替えを管理します。

**使用方法**:
1. サイドバーから「雨天時モード管理」をクリック
2. 雨天モードのON/OFFを切り替え
3. 雨天時の競技スケジュールを確認・調整

---

### 2.3 競技情報登録・管理
**パス**: `/dashboard/root/sport-management`

**機能概要**: 大会で実施する競技の登録・編集・削除を行います。

**使用方法**:
1. サイドバーから「競技情報登録・管理」をクリック
2. 「新規競技追加」から競技を登録
3. 競技名、ルール、開催日時、場所などを設定

**設定項目**:
- 競技名
- 競技種別（個人/団体）
- 参加対象学年
- 開催日時・場所
- ポイント配分

---

### 2.4 通知管理
**パス**: `/dashboard/root/notification`

**機能概要**: 全ユーザーへのプッシュ通知を管理・送信します。

**使用方法**:
1. サイドバーから「通知管理」をクリック
2. 「新規通知作成」でタイトルと本文を入力
3. 送信対象（全体/特定の学年/特定のクラス）を選択
4. 「送信」ボタンで即時送信、または予約送信を設定

---

### 2.5 通知申請管理
**パス**: `/dashboard/root/notification-requests`

**機能概要**: 学生からの通知申請を承認・却下します。

**使用方法**:
1. サイドバーから「通知申請管理」をクリック
2. 申請一覧から内容を確認
3. 「承認」または「却下」を選択
4. 承認された通知は自動的に全体へ送信

---

### 2.6 トーナメント生成・管理

**使用方法**:

2. 対象競技を選択
3. 「トーナメント生成」でランダムにチームを配置
4. 必要に応じて手動で配置を調整
5. 「確定」で公開

---

### 2.7 昼競技管理

**機能概要**: 昼休みに行われる特別競技の設定・管理を行います。

**使用方法**:

2. 昼競技の種目、参加者、ルールを設定

### 2.8 権限管理
**パス**: `/dashboard/root/user-promotion`

**機能概要**: ユーザーに `admin` / `root` の基本権限を付与・剥奪します。

**使用方法**:
1. サイドバーから「権限管理」をクリック
2. メールアドレスまたは表示名でユーザーを検索
3. 対象ユーザーの `admin` または `root` 列から「付与」または「剥奪」を実行

**注意**:
- この画面で変更できるのは `admin` / `root` の基本権限のみです
- 任意ロールの付与・削除は Admin の「ロール管理」から行います

## 3. Admin機能（Admin/Root共通）

Admin権限以上のユーザーがアクセスできる機能です。

### 3.1 クラス・チーム割り当て
**パス**: `/dashboard/admin/class-management`

**機能概要**: 生徒を各競技のチームに割り当てます。

**使用方法**:
1. サイドバーの「Admin」セクションから「クラス・チーム割り当て」をクリック
2. 競技を選択
3. クラスと参加者を選択してチームに割り当て
4. 「保存」で確定

**備考**: 一般ユーザーはプロフィールで設定した自クラスのみを対象に操作できます。Admin/Rootは任意のクラスを対象に操作できます。重複登録の上限を超える選手が含まれる場合は登録できません。上限は大会情報登録・管理でrootが設定します。

---

### 3.2 ロール割り当て・管理
**パス**: `/dashboard/admin/role-management`

**機能概要**: 審判ロールなど、イベント運営で必要になる任意ロールの付与・削除を行います。

**使用方法**:
1. サイドバーから「ロール割り当て・管理」をクリック
2. ユーザーを検索
3. 任意のロールを付与または削除

**注意**:
- `root` / `admin` / `student` の基本権限はこの画面では変更できません
- クラス単位の操作対象はロールではなくプロフィールのクラス所属で決まります

---

### 3.3 バーコード読み取り
**パス**: `/dashboard/admin/barcode-reader`

**機能概要**: 学生証等のMyIDバーコードを読み取り、参加本登録とラウンドチェックインを行います。

**使用方法**:
1. サイドバーから「バーコード読み取り」をクリック
2. 競技と試合を選択
3. カメラでMyIDバーコードを読み取る、またはバーコード値を手入力
4. 読み取り成功で参加本登録とラウンドチェックインを完了

---

### 3.4 参加本登録確認
**パス**: `/dashboard/admin/confirmed-participants`

**機能概要**: MyIDバーコード読み取りで登録済みの参加者一覧を確認します。

**使用方法**:

2. 競技・日時で絞り込み
3. 登録済み参加者の一覧を確認

---

### 3.5 試合結果入力

**機能概要**: 各試合の結果（スコア・勝敗）を入力します。

**利用条件**: 対象大会のステータスが「開催中」の場合のみ結果を入力できます。準備中・予定・アーカイブでは入力できません。

**使用方法**:
1. サイドバーから「試合結果入力」をクリック
2. 競技とマッチを選択
3. 各チームのスコアを入力
4. 「結果確定」で保存

---

### 3.6 出席登録

**機能概要**: 競技への出席状況を管理します。

## 補足情報

### プッシュ通知について
- Student、Admin、Root権限を持つユーザーがプッシュ通知を受信できます
- ブラウザの通知許可が必要です

### セキュリティについて
- 各機能は権限に応じてフロントエンド（画面表示）とバックエンド（API）の両方で保護されています
- 権限のない機能にアクセスしようとすると、ダッシュボードにリダイレクトされます

### 権限の階層
```
Root > Admin > Student
```
上位の権限は下位の権限の機能全てにアクセスできます。
