# Security Policy

## 脆弱性の報告

脆弱性を見つけた場合は、公開 Issue、公開 Pull Request、公開 Discussion には書き込まないでください。

代わりに GitHub Security Advisory を使って、非公開で報告してください。

1. このリポジトリの **Security** タブを開く
2. **Report a vulnerability** を選択する
3. 影響範囲、再現手順、想定されるリスクを記入する
4. 必要に応じて PoC やログを添付する

公開された場所に脆弱性情報が投稿された場合、被害を防ぐため、その投稿を削除または非公開化することがあります。

## 報告に含めてほしい内容

- 影響を受ける機能、画面、API
- 再現手順
- 期待される挙動と実際の挙動
- 影響度の見立て
- 修正案がある場合はその概要

個人情報、認証情報、秘密鍵、Cookie、トークンなどは、必要最小限にマスクしてください。

## 対応方針

報告を受け取ったら、内容を確認し、影響範囲と優先度を判断します。修正が必要な場合は、非公開の Advisory 内で調整し、修正後に必要な範囲で公開します。

脆弱性の詳細は、修正が利用可能になるまで公開しないでください。

## サポート対象

現在サポート対象として扱うのは、`main` ブランチの最新状態です。
